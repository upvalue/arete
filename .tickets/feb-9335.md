# Bignum / arbitrary-precision integer support

Arete silently overflows fixnum arithmetic (verified: (expt 2 100) => 0; (* 1000000000 1000000000) => 10^18 only because it still fits in a 64-bit fixnum). pi and chudnovsky both depend on bignum arithmetic: pi's first step is (expt 10 nb-digits) with nb-digits=50, which overflows to 0 and triggers SIGFPE on subsequent quotient. chudnovsky returns all-zero output for the same reason. exact-integer-sqrt has also been shimmed (feb-uotk) but cannot be accurate past fixnum range without bignums underneath. Scope is large — touches src/numbers.cpp plus the reader and likely the GC. Likely deferred but tracked.

## Acceptance Criteria

Either bignums are implemented (pi and chudnovsky PASS or TIMEOUT), or a decision to skip these two benchmarks is recorded in docs/Benchmarks ecraven.md.
