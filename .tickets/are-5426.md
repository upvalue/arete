# Exp 2: inline small memq/memv/assq

Autoresearch experiment: inline (memq/memv/assq X 'LIT) for short (<=4) quoted literal second arg. Rewrite to an if-chain of eq? tests, with memv gated on safe-for-eq? elements (symbols/fixnums/booleans/'()) to avoid the char/flonum boxing pitfall noted in arete_char_eq_inline_bug memory.

Grep of literal-list patterns in the three target benchmarks:
  browse.scm:  0 memq/memv/assq-with-'(...) uses
  nboyer.scm:  0 (all assq calls use dynamic alists: *symbol-records-alist*, subst)
  peval.scm:   1 (line 180: memq (const-value (cadr exp)) '(#f ()))

peval.scm's hot path is the env-lookup assq-over-dynamic-alist + the partial-eval recursion, not the single literal-list memq. So expected negligible impact, observed same. Boot.scm's compiler/expander/syntax library has several literal-list memqs that DO get inlined at bootstrap compile time, but that only affects pre-bench compile time, which is outside the benchmark window.

Lessons / notes for next agent:
- The implementation rewrites (memq X '(a b c)) to ((lambda (tmp) (if (eq? tmp 'a) '(a b c) ...)) X) and dispatches through compile-inline-call, which cleanly gets the inline-lambda treatment when not at toplevel. Constants are shared via register-constant's linear dedup (tails don't get hash-table sharing, but length <=4 × few call sites means tiny constant-vector cost).
- Watch out for (cadr X) vs (caddr X) when extracting the 2nd argument of a call form — (cdr x) is the args list, so (cadr x) is the FIRST arg; the quote-form is (caddr x). Initial implementation fired but crashed trying to destructure the arg as if it were the quote form.
- Bench noise-floor hit immediately; the optimization doesn't target the actual hotspots.
- If a future agent wants to extend: the bigger wins are (a) inline assq when the alist has string keys (not currently safe-for-eq? but could work with string=? chain), (b) detect and specialize (cond ((memq x '...)) ...) which is a common pattern but peval-style code uses `case` (already handled by %member-impl via memv, mostly short) — could try an explicit `case` lowering that bypasses memv entirely.
